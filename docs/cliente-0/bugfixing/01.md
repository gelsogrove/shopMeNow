## BUG-01 | Welcome doppio + lingua sbagliata (20 Apr 2026)

**Symptom:**
```
Ciao non mi va la lavatrice
→ ¡Hola! 👋 Soy EcoLaundry, tu asistente de Ecolaundry.       ← welcome in spagnolo
Ciao non mi va la lavatrice                                     ← messaggio utente DUPLICATO in storia
→ ¡Hola! 😊 Para poder ayudarte mejor, ¿puedes decirme en qué local te encuentras? 🏪
Goya
→ Hola Nuevo Cliente, te estoy conectando con nuestro equipo de mantenimiento...
```

**Root Causes:**
1. `ultramsg-webhook.controller.ts`: chiamava `WelcomeMessageHandler` nel webhook, salvava `[user:msg]` + `[assistant:welcome]` + settava flag atomico `welcomeSent=true`. Poi `ChatEngine` non trovava più `welcomePrefix`, ma il template aveva `{{welcomeMessage}}` → LLM riscriveva il welcome nel body → DB riceveva `[user:msg DUPLICATO]` + `[assistant:welcome+domanda]`. La storia duplicata faceva classificare "Goya" come CONTRADDIZIONE → `contactOperator`.
2. `00-router.template.md`: conteneva la sezione `## WELCOME / {{welcomeMessage}}` che faceva iniettare il welcome nel prompt LLM.
3. `sync-ecolaundry-flow.ts`: backtick non escapato (`` `explicitRequest: true` ``) rompeva il template literal TypeScript → crash su Heroku.
4. `seed.ts`: stesso backtick non escapato.

**Fix applicati:**
- `ultramsg-webhook.controller.ts`: per workspace FLOW, WelcomeMessageHandler skippato → ChatEngine gestisce welcome+response in un unico messaggio via `welcomePrefix`
- `00-router.template.md`: rimossa sezione `## WELCOME` con `{{welcomeMessage}}`
- `sync-ecolaundry-flow.ts` + `seed.ts`: escapato backtick → `\`explicitRequest: true\``

**Status:** ✅ Fix deployato — sync script eseguito su Heroku

## BUG-03 | "Nuovo Cliente" nel saluto (20 Apr 2026)

**Symptom:**
```
Hola Nuevo Cliente, te estoy conectando...
```
Il bot saluta con "Nuovo Cliente" invece di omettere il nome quando non è registrato.

**Root Cause:** `humanSupportInstructions` nel seed contiene `{{nameUser}}` che viene sostituito con il valore del DB. Se il cliente non è registrato, il nome è "Nuovo Cliente" (default).

**Fix da fare:** Commentare/rimuovere `{{nameUser}}` dal `humanSupportInstructions` in Settings o gestire il fallback — NON cancellare la logica, solo commentare il saluto quando nome = default.

**Status:** ⏳ Da fare (issue aperta)

## BUG-04 | contactOperator senza messaggio quando hasHumanSupport=false (20 Apr 2026)

**Symptom:** Il bot chiama `contactOperator` (o decide di escalare) ma il cliente non vede nessun messaggio — chatbot si disattiva silenziosamente.

**Root Cause:** `flow-workspace.strategy.ts` mostrava `humanSupportInstructions` SOLO quando `workspace.hasHumanSupport !== false`. Ma Ecolaundry ha `hasHumanSupport: false` → messaggio non veniva mai mostrato.

**Fix:** `humanSupportInstructions` ora viene sempre mostrato quando `shouldCallOperator=true`, indipendentemente da `hasHumanSupport`.

**Status:** ✅ Fix deployato

## BUG-05 | changeLanguage chiamata su parola singola / nome città (19 Apr 2026)

**Symptom:** Il bot cambiava lingua quando l'utente scriveva "Goya" (nome lavanderia) o parole tecniche straniere.

**Root Cause:** LLM chiamava `changeLanguage()` troppo liberamente — nessun guard su `explicitRequest`.

**Fix:**
- `changeLanguage.ts`: aggiunta guard `if (!request.explicitRequest) return { success: false, error: "NOT_EXPLICIT_REQUEST" }`
- Router prompt: aggiunta sezione `## LANGUAGE CHANGE RULE` con regola: chiamare solo con `explicitRequest: true` e SOLO per frasi esplicite di cambio lingua

**Status:** ✅ Fix deployato

## BUG-06 | CONFIRMATION loop (bene/fatto → re-prompt × 4 → escalate) (19 Apr 2026)

**Symptom:** Su nodi CONFIRMATION, risposte tipo "bene", "fatto", "hecho" venivano classificate come AMBIGUOUS → re-prompt × 4 → escalate operatore.

**Root Cause:** `flow-engine.service.ts handleAmbiguous()` non gestiva CONFIRMATION nodes → cascata nel loop generico.

**Fix:** `flow-engine.service.ts`: prima del fallback generico, branch per nodi CONFIRMATION (type=CONFIRMATION o transitions YES/NO) → restituisce `isAmbiguousChoice: true` → Sub-LLM classifica correttamente YES/NO.

**Status:** ✅ Fix deployato
