## SESSION ANALYSIS (2025)

### Root Cause Investigation

Test message: `"Ciao non parte l'asciugatrice"` (first message from new customer)

**Observed sequence (BUGGY)**:
1. Bot sends: "Ciao! 👋 Sono EcoLaundry, il tuo assistente Ecolaundry."  ← welcome only, no machine handling
2. User repeats: "Ciao non parte l'asciugatrice"  ← user forced to repeat
3. Bot: "Mi dispiace sapere che l'asciugatrice non parte... In quale locale ti trovi?"
4. User: "Roma"
5. Bot: immediately calls `contactOperator()` ← NO clarifying question

---

## BUG-W1: Welcome separate + user must repeat

**Root cause**: The OLD router prompt classified `"Ciao non parte l'asciugatrice"` as **GREETING** intent  
→ returned ONLY the welcome bubble, ignoring the machine problem  
→ user had to repeat their message before the bot would act

**Fix applied**: New `ECOLAUNDRY_ROUTER_PROMPT` includes INTENT CLASSIFICATION:
- `GREETING` → welcome ONLY, do NOT ask anything
- `MACHINE_PROBLEM` → acknowledge in one sentence + start gather flow

With new prompt: "Ciao non parte l'asciugatrice" → MACHINE_PROBLEM → bot answers:  
`[welcome prefix] + "Capito, non preoccuparti! 🌀 In quale dei nostri locali ti trovi?"` → **ONE combined message**

**Status**: ✅ FIXED — deployed via `sync-ecolaundry-flow.ts` (router `systemPrompt` updated in DB)

**Files changed**:
- `apps/backend/src/templates/flow/00-router.template.md` — updated to match new DB prompt (rule #16)
- `packages/database/scripts/sync-ecolaundry-flow.ts` — was the source of the new prompt

---

## BUG-W2: City name "Roma" → immediate contactOperator

**Root cause**: OLD router prompt had no explicit handling for unrecognized locations  
→ when user said "Roma" (not a known Ecolaundry branch), bot immediately escalated

**Fix applied**: New `ECOLAUNDRY_ROUTER_PROMPT` GATHER FLOW step 1 now says:
```
If the customer's answer does NOT match any known branch (e.g. "Roma" or unrecognised answer),
do NOT escalate — ask ONE clarifying question: "Sorry, could you tell me which self-service laundry you are in?"
(max 2 clarifications). Only after 2 failed attempts escalate to contactOperator.
```

**Status**: ✅ FIXED — deployed via `sync-ecolaundry-flow.ts`

---

## BUG-W3: "Si ho gia' pagato" → wrong semantic transition in washer flow

**Root cause** (subtle):
1. User is at the `caso_importo` CHOICE node (display shows a price)
2. OLD question: "did the central unit return your coins or change?"
3. `flow-classifier.service.ts` extended MATCH regex: `/^(s[iì]|yes|ok|no|nope)(?=\s|[,.!])/i` → "Si ho gia' pagato" → **MATCH → YES**
4. `applyTransition` → `normalizeInput` → "YES" → route to `cambio_si`
5. `cambio_si` said: "since the central unit returned your change, re-insert the full amount"
6. **SEMANTIC MISMATCH**: "I already paid" ≠ "machine returned coins to me" — wrong path!

**Fix applied**: Rephrased `caso_importo` question to ask about PAYMENT (not about change return):
- New question: "Did you already pay at the central unit and it still shows this price, or have you not paid yet?"
- YES (I already paid) → now routes to `cambio_no` (check if credit is on another machine number) ✅ CORRECT
- NO (haven't paid yet) → routes to `cambio_si` (guide through payment) ✅ CORRECT

**Status**: ✅ FIXED — deployed via `sync-ecolaundry-flow.ts` (washer `flowJson` patched in DB)

**Files changed**:
- `packages/database/prisma/seed.ts` — `caso_importo`, `cambio_si`, `cambio_no` nodes updated
- `packages/database/scripts/sync-ecolaundry-flow.ts` — washer flowJson patch + caso_importo fix
