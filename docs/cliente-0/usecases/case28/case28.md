# Case 28 — Relato contradictorio en un doble cobro

## SCENARIO

**Scenario 28.1 — Escalation:** El cliente da un relato confuso y contradictorio (ej. "Me cobró dos veces, aunque creo que pagué en efectivo, pero no sé si arrancó"). El chatbot reconoce la incoherencia y escala sin discutir la contradicción.

## ACCEPTANCE CRITERIA

- El bot reconoce incoherencia sin confrontar.
- El bot escala para revisión manual.
- Tono empático y tranquilo.

## CONVERSATION — Case 28 — Scenario 28.1

**Usuario:** Me cobró dos veces, aunque creo que también pagué en efectivo, pero no sé si llegó a arrancar.
**Bot:** Gracias. Vamos a revisarlo contigo. ¿Has podido finalmente usar el servicio?

**Usuario:** No lo sé bien.
**Bot:** De acuerdo. Para comprobarlo bien, vamos a revisar el caso manualmente. ¿Cómo te llamas?

**Usuario:** Jordi.
**Bot:** Gracias Jordi. Un operador revisará tu caso a la mayor brevedad posible. ¿Aceptas recibir una llamada telefónica?

**👤 Human Support message**
Usuario Jordi reporta relato confuso de doble cobro: ¿dos pagos con tarjeta?, ¿pago en efectivo?, ¿máquina arrancó?. Requiere investigación manual de transacciones y máquina.

## REPORT LLM

- No se detectaron criterios negativos.
